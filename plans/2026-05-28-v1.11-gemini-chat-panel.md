**Date:** 2026-05-28
**Status:** Approved
**Plan author:** Design Agent (via Orchestrator), informed by Research Agent findings
**Approved by:** Tom (approved via plan review; model choice = Gemini Flash, quick-answer tuned)

## The Idea, In Plain English

A floating chat bubble (bottom-right) opens a slide-over "Ask Gemini" panel inside StartDashboard. Tom types a question and gets a concise, search-style answer without leaving the portal. A header "Open full Gemini ↗" button links out to gemini.google.com when he wants his real account. Powered by the free Gemini API (Google AI Studio key, per-browser localStorage). Global assistant (available from every persona).

## Why Gemini Flash

Research confirmed gemini.google.com cannot be iframed (X-Frame-Options: DENY), so a native API-backed chat is the path. Gemini Flash is free (no credit card), and it's the same model family behind Google Search's AI answers Tom already reaches for. Tuned via system instruction for concise quick answers. Claude/OpenAI have no free tier; Gemini fits the "fast quick answer" use case best.

## What It Will Do
- Floating, theme- and accent-aware chat button (bottom-right), available from every persona.
- Slide-over panel: header (title, model label, "Open full Gemini ↗", close ✕), scrollable history, input (Enter sends, Shift+Enter newline), "Clear chat".
- Calls `https://generativelanguage.googleapis.com/v1beta/models/<model>:generateContent` (non-streaming, default model = current free Flash) with `x-goog-api-key` header.
- `system_instruction` primes concise, direct, search-style answers.
- Sends last ~10 turns for follow-up context.
- Chat history in localStorage PER-DEVICE (not synced to gist — avoids transcript bloat).
- "Open full Gemini ↗" → gemini.google.com (works even with no key).
- Settings: "Gemini API key" field + inline walkthrough (link to aistudio.google.com/app/apikey, free/no card) + optional model-override field. Non-destructive save (empty = no change), per the v1.8 rule.
- Empty state (no key): "Add your free Gemini API key in Settings to chat here"; link-out still works.
- Friendly error handling for 429 / network / CORS — message, not a stack trace; link-out always available.
- Works in light + dark mode; does not disturb widgets/categories layout.

## What It Will NOT Do
- No iframe embed of gemini.google.com (impossible).
- No connection to Tom's personal Gemini account history (fresh assistant; link-out is for his account).
- No streaming in v1 (non-streaming; avoids streaming-endpoint CORS wrinkle).
- No per-persona priming in v1 (global; easy future add).
- Chat history not synced across devices (per-device by design).
- No new backend (direct browser call; zero-backend architecture preserved).

## Approach / Key implementation notes
- New `LS.GEMINI_KEY='sd_gemini_key'`, `LS.GEMINI_MODEL='sd_gemini_model'`, `LS.CHAT_HISTORY='sd_chat_history'`. `state.geminiKey/geminiModel/chatHistory` from localStorage on boot.
- Default model string: use the current free Flash model (e.g. `gemini-2.5-flash`); model-override field lets Tom change it without a code edit if Google renames.
- Request body: `{ system_instruction: { parts:[{text: <concise-answer prompt>}] }, contents: [ ...history as {role:'user'|'model', parts:[{text}]} ] }`. Parse `candidates[0].content.parts[0].text`.
- CORS: non-streaming `:generateContent` with API key generally works from the browser; Build Agent verifies empirically. If blocked, graceful fallback = clear message + link-out; do NOT add a backend.
- Panel/button styled from existing CSS theme tokens so light/dark both work; accent from current persona.
