# Build Plan: StartDashboard v1.12 — Weather forecast detail

**Date:** 2026-05-28
**Status:** Approved
**Plan author:** Design Agent (via Orchestrator)
**Approved by:** Tom (chose: condition icon + label, and chance of rain %, per forecast day)

## The Idea, In Plain English

The Weather widget's forecast days currently show only high/low temp. Add a weather condition icon + short label (Sunny, Rain, Partly cloudy…) and a chance-of-rain percentage to each forecast day. Also add the condition icon to the current-conditions line so it's consistent.

## What It Will Do
- Each forecast day shows: condition glyph (☀ ⛅ ☁ 🌧 ❄ etc.) + short label + high/low temp + chance of rain (%).
- Current conditions line gains the matching condition glyph next to the temp.
- All from Open-Meteo (free, no key). No new dependencies.

## What It Will NOT Do
- No wind / UV (Tom didn't select them this round — easy future add).
- No hourly breakdown; still a 3-day forward forecast.
- No layout overhaul — same compact day cells, just richer content.

## Files to Be Changed
| File | What |
|------|------|
| `projects/personal-dashboard/index.html` | Weather fetch + render: add `precipitation_probability_max` to the daily query; new `weatherCodeIcon(code)` glyph map; render icon+label+precip per forecast day; add icon to current line |

## Approach
- In the Open-Meteo fetch (`fetchWeather` / the weather widget fetch), add `precipitation_probability_max` to the `daily=` params. `weather_code` is already fetched daily.
- Add `weatherCodeIcon(code)` returning an emoji glyph, paralleling the existing `weatherCodeText(code)` label map (reuse that for labels).
- Update `renderWeatherWidget` forecast-day markup to include glyph + label + precip%. Keep cells compact; wrap gracefully on narrow widths.
- Theme-token styling; works light/dark.

## Dependencies
None new. Open-Meteo only, still no key.

## Open Questions for Tom
None.
