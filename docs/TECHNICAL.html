Architecture Overview

How the app is structured and how the pieces fit together.

StartDashboard is a single HTML file. Everything — markup, styles, and behavior — lives in index.html. There is no build step, no framework, and no server. The page is served as a static asset (GitHub Pages in production; you can also double-click the file locally).

The app has three runtime layers:

  • State. A single state object holds everything: the GitHub token (read from localStorage on boot), the gist ID, the device's default persona, and the synced state.data object that mirrors the gist's JSON. All UI rendering reads from this object; all mutations go through a small setState() wrapper that re-renders and triggers a debounced save.
  • Sync. Reads and writes go to the GitHub REST API at api.github.com/gists/:id. Loading is a single GET; saving is a single PATCH that replaces the JSON file's content. Saves are debounced ~1.5 seconds after the last edit so rapid changes coalesce. A 60-second background poll checks updated_at on the gist and surfaces a "stale" indicator if another browser updated it.
  • Render. Vanilla DOM manipulation. The whole dashboard re-renders on persona switch (it's cheap — there are few enough elements). Per-widget render functions handle clock (re-rendered every 30 seconds), weather (fetched from Open-Meteo on render), todos, and the notes textarea.

The bookmarklet adds bookmarks via a separate code path. When the URL contains ?add=<url>&title=<title>, the page renders only a compact "Pick a persona + category" overlay, writes synchronously to the gist, and closes its window. The main dashboard is never instantiated in that mode.

Key Decisions

Decisions made during the build, with the reasoning preserved so future-Tom understands why.

GitHub gist as the data backend, not a third-party service

StartMe-style services bring ads and lock-in. Supabase or jsonbin work but add a dependency on a free tier that may go away. GitHub gists are durable as long as Tom's GitHub account exists, free, and use auth Tom already understands.

decided 2026-05-27 during plan revision

Personal Access Token per browser, stored in localStorage

Avoids the complexity of OAuth flows in a single static page. Each browser gets its own token; if a device is lost, only that one token needs to be revoked. The token is sent only to api.github.com.

decided 2026-05-27

Per-persona widgets, backgrounds, and accent colors

The whole point of personas is mental separation between Work and Personal. Sharing widgets across personas would blur that. Each persona is fully independent.

decided 2026-05-27 after Tom raised the persona concept

Per-device default persona via localStorage (not in the gist)

Tom wants his work Mac to open to Work and his iPhone to open to Personal. The default persona is a per-device preference, not a synced setting. Stored as sd_default_persona in localStorage.

decided 2026-05-27

Bookmarklet over a browser extension

A browser extension means three separate builds (Chrome, Safari, Firefox) and ongoing maintenance as their extension formats evolve. A bookmarklet is one piece of JavaScript that works in every browser, forever.

decided 2026-05-27 during planning

Backgrounds: 8 CSS-only presets + image URL only (no upload in v1)

Image upload would mean storing base64 in the gist, which bloats the JSON on every save. URL-based images work well and keep the gist small. Upload can be added later if Tom wants it.

decided 2026-05-27

Last-write-wins on sync conflicts

Real conflict resolution is overkill for a single-user app. If Tom edits in two browsers simultaneously, a toast warns him and the most recent save wins. The 60-second background poll surfaces stale state proactively.

decided 2026-05-27

Dependencies & Setup

How to set up this project on a fresh machine. Every install, every config file, every environment variable.

There are no local dependencies. No npm install, no requirements.txt, no Docker. The project is one HTML file.

External services Tom must set up once

  • GitHub repository at github.com/TECGUNTHER/startdashboard with GitHub Pages enabled (Settings → Pages → main / root).
  • GitHub Personal Access Token with gist scope. Generated once per browser. Pasted into the dashboard's first-run setup screen.
  • One private gist on Tom's GitHub account. Created automatically by the dashboard on first-run setup if Tom picks "First browser — create a new gist".

Per-browser localStorage keys (set during first-run setup)

  • sd_pat — GitHub Personal Access Token (this browser only)
  • sd_gist_id — ID of the gist storing this dashboard's data
  • sd_default_persona — ID of the persona this device opens to by default

How to Run

The launch command, the port number, and common troubleshooting.

To launch (production)

open https://tecgunther.github.io/startdashboard/

To preview locally (no sync)

Just double-click index.html, or:

open projects/personal-dashboard/index.html

The setup screen will appear since no PAT is in localStorage; you can paste a token to test sync locally if you want.

Troubleshooting

  • "Bad credentials" on first run — PAT is wrong or missing the gist scope. Regenerate at github.com/settings/tokens.
  • Sync indicator turns red ⚠ — click it to reload. If still failing, the token may have expired; reset this browser in Settings.
  • Cross-browser changes not showing — reload the other browser (Cmd+R). Background poll is 60s, but reload is instant.
  • Bookmarklet popup blocked — allow popups for tecgunther.github.io in browser settings.
  • Weather "Could not load" — transient Open-Meteo issue, or invalid lat/lon. Confirm coordinates in Settings.

How to Modify

"If you want to change X, look in Y." A quick map for future edits.

  • Add a new background preset. Add a CSS class .bg-<name> in the <style> block of index.html (search for "Background presets") and add a matching entry to BG_PRESETS in the script.
  • Add a new widget. Add a <div class="widget"> entry to the widgets-row section, then write a render<Name>Widget(persona) function and call it from renderWidgets().
  • Change the dark color palette. CSS custom properties live at the top of the :root block in index.html.
  • Change the gist filename. Constant GIST_FILENAME in the script. Changing this on an existing setup will silently break sync — only change before first deploy.
  • Disable the favicon service for privacy. Search for s2/favicons in index.html and either remove the block or replace with your own service.
  • Change the search engine list. The SEARCH_ENGINES object near the top of the script holds URL templates.
  • Change the bookmarklet popup size. The bookmarklet JavaScript is generated in openBookmarkletModal(); the width=520,height=600 values are inline there.
  • Adjust the save debounce. The 1500ms timeout in setState().
  • Adjust the stale-check interval. The 60000ms interval at the very bottom of the script.

Known Limitations

What the project doesn't do, what it can't do without significant work, what's brittle.

  • Real-time collaboration is not supported. Sync polls every 60 seconds and saves are debounced ~1.5 seconds. If two browsers edit at once, last save wins and a toast surfaces the conflict.
  • Image uploads are not supported in v1. Backgrounds can be presets or URL-based images. Adding upload would mean base64-encoding into the gist, which bloats it on every save.
  • No automatic bookmark import. Bookmarks come over manually as Tom uses the bookmarklet. Adding a JSON-import flow is straightforward if needed later.
  • PAT lives in localStorage. Anyone with physical access to an unlocked browser could read it. For a personal-use app on a personal machine this is acceptable. For shared devices, log out / lock the screen.
  • Gist is private but not encrypted. GitHub staff and GitHub's automated systems could, in principle, see the contents. The data is bookmarks and notes — not secrets — so this is acceptable.
  • The favicon service sends bookmark domains to Google. If this matters to you, see "How to Modify" above.
  • Bookmarklet popups can be blocked. Most browsers allow them for the originating site by default, but some configurations block them entirely. Tom can allow popups for tecgunther.github.io if it happens.
  • No undo. Delete a bookmark and it's gone. Worth adding a "Recently deleted" trash bin in a future iteration.
