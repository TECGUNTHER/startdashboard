How to Run This App

Key URLs

  • Live dashboard: https://tecgunther.github.io/startdashboard/
  • GitHub repo (for code edits): github.com/TECGUNTHER/startdashboard
  • GitHub Pages settings: github.com/TECGUNTHER/startdashboard/settings/pages
  • Personal Access Tokens (to revoke or regenerate): github.com/settings/tokens
  • Find your gist ID: open the dashboard → Settings (⚙) → Sync → Gist ID

One-time setup (already done)

The project is deployed and the first browser is set up. To repeat this whole sequence on a fresh machine, see README.md.

git init && git add . && git commit -m "Initial StartDashboard"
git remote add origin https://github.com/TECGUNTHER/startdashboard.git && git push -u origin main

Then in GitHub: Settings → Pages → Deploy from a branch → main / (root).

To launch

Open in any browser:

open https://tecgunther.github.io/startdashboard/

For local-only preview (no sync), just double-click index.html.

No port required — static page served by GitHub Pages.

Troubleshooting

  • "Bad credentials" on first run · PAT was pasted wrong or is missing the gist scope. Regenerate.
  • Sync indicator turns red · Click it to reload from the gist. If still failing, the token may have expired — regenerate and reset this browser in Settings.
  • Bookmarklet popup blocked · Allow popups for tecgunther.github.io in browser settings.
  • Changes not appearing across browsers · Reload the other browser. Sync polls every ~60s; reload is instant.

2026-05-28 · Thursday
v1.3 hotfix — the background bug that was lurking since v1.0, plus clock zone duplicates and a duplicate Settings UI.

Completed

  • Background rendering bug fixed (CSS specificity). #background { background: var(--bg) } was overriding every .bg-* class rule due to ID-vs-class specificity. Result: every preset rendered as the same solid dark color since v1.0. Tom never actually saw Midnight Mesh, Aurora, Synthwave, etc. — they exist now. Removed the conflicting rule.
  • Clock zone duplicates bug fixed. Zones were stored as IANA strings, so adding "Atlanta" silently failed if NYC was already there (same TZ). Migrated to { name, tz } objects so two cities sharing a timezone render as separate rows. Backward-compat: old string-format zones auto-migrate on load.
  • Removed duplicate Clock/Weather config from main Settings. The original "comma-separated IANA names" and "Name, lat, lon" fields competed with the widget-gear search UIs added in v1.1. One way to configure each widget now: its ⚙ icon. Main Settings panel got a one-line pointer instead.
  • saveSettings now flushes immediately (await instead of relying on the 1.5s debounce). Fast reloads no longer risk missing the save.

Decisions made

  • Schema migration is invisible to Tom — old gists with string zones load cleanly and re-save in the new shape on first edit.
  • Clock allows duplicate timezones with different city labels (Atlanta + NYC both EST is fine); de-dupe only on exact name + tz match.
2026-05-27 · Wednesday · late evening
v1.2 expansion — per-persona widget visibility, three finance widgets, 3 bug fixes.

Completed

  • Per-persona widget visibility. Each persona can independently show/hide any widget via Settings ⚙. Default: Work and Personal hide the new finance widgets; new personas show everything.
  • Crypto widget (CoinGecko, no API key). Search any coin by name or symbol; shows price + 24h % change.
  • Markets widget (Twelve Data). Defaults to SPY / QQQ / DIA for S&P / Nasdaq / Dow. Pretty-prints friendly names.
  • Watchlist widget (Twelve Data). Add any stock by ticker or via search.
  • 60-second client cache on Twelve Data quotes so the free 8-req/min limit isn't burned by re-renders.
  • Bug fix: 12-hour clock (was 24-hr).
  • Bug fix: background save was silently failing if no thumbnail was clicked in settings — now reads the current background as fallback and properly marks the pre-selected thumb.
  • Bug fix: cross-add banner Switch button now reloads the gist before switching personas, so the just-added bookmark appears immediately (no manual refresh needed).
  • File grew from ~2,860 to ~3,360 lines. Still single file, still no framework.

Decisions made

  • Twelve Data API key stored per-browser in localStorage (parallel to GitHub PAT) — rate limits are per-key, and we shouldn't sync credentials through the gist.
  • Markets uses ETF proxies (SPY, QQQ, DIA) for indices — the actual indices (^GSPC etc.) sometimes need higher Twelve Data tiers.
  • Crypto uses CoinGecko's free public API directly — no key, no rate-limit headaches for personal use.
  • Widget visibility is per-persona, not global — matches the persona-as-channel mental model.
  • For Markets/Watchlist, the search input also supports "type ticker, press Enter" so tickers Twelve Data search doesn't surface (rare indices) can still be added directly.
