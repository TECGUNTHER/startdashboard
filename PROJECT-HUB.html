  • v1.19 shipped — fixed category cards vanishing when you used the move arrows. A bug slipped in with v1.18: clicking the ▲▼ arrows (or “Move to front/end”) on a category (bookmark) card made the whole card disappear, and “Reset layout” couldn’t bring it back — only a full browser refresh would. Cause: the one shared helper that reorders cards was putting back the card’s id label instead of the card itself. Widgets are just labels, so they were fine; categories are full objects (title, bookmarks, color), so they got wiped out and the card had nothing left to draw. The fix keeps the real card when moving it. No data was ever lost — your categories were always safe in GitHub; the breakage only ever lived in the browser’s temporary copy, which is why a refresh restored them. Only index.html changed; no new dependencies.
  • v1.18 shipped — rearranging cards is finally reliable. Three fixes to how you move widget and category cards around. First, the drag no longer blinks. The old drag inserted a real placeholder card into the layout on every tiny mouse move, which forced the whole Pinterest-style masonry to re-pack each time — that constant re-packing was the flickering you saw. Now, while you drag, a card-sized slot is no longer shoved into the flow; instead a single glowing edge bar lights up on the left or right side of whichever card you’re hovering, showing where the card will drop. Nothing in the layout moves until you let go, so there’s no reflow and no blink. Second, the ▲▼ move arrows are now always visible (faintly, brightening on hover) on every widget and category — they’re promoted to the primary, precise way to rearrange, and each card’s menu also gained “Move to front” / “Move to end” for big jumps. Third, Settings has a new “Reset layout” button that puts the widgets back to their default order in one click if things ever get messy. Honest note: because masonry packs cards by height, a dropped card lands approximately where you aimed, not pixel-perfect — the arrows and Reset are the precise tools, and the de-flickered drag is the quick-and-rough one. Only index.html changed; no new dependencies.
  • v1.17 shipped — the Ask Gemini chat got a 🎤 voice button and now shows formatted answers. Two upgrades to the in-dashboard Gemini chat. First, a 🎤 microphone button sits in the chat input row (between the text box and Send): click it, speak your question, and it’s typed into the box for you to review and send — it never auto-sends, and it never wipes text you already typed. It uses your browser’s built-in speech-to-text (the same engine behind dictation), so it’s free, needs no key, and works on iPhone Safari too. On any browser that doesn’t support voice, the button simply doesn’t appear — nothing breaks. If you deny microphone permission, you get a friendly note rather than an error. Second, Gemini’s replies now render as proper formatting — bold, italics, bullet and numbered lists, headings, inline and block code, and clickable http(s) links — instead of showing the raw markdown characters. The formatter is built in-house and security-first: it escapes everything first and only ever inserts a small whitelist of safe tags, so nothing in a model reply can run code or inject HTML. How to Run This App

Key URLs

  • Live dashboard: https://tecgunther.github.io/startdashboard/
  • GitHub repo (for code edits): github.com/TECGUNTHER/startdashboard
  • GitHub Pages settings: github.com/TECGUNTHER/startdashboard/settings/pages
  • Personal Access Tokens (to revoke or regenerate): github.com/settings/tokens
  • Finnhub signup (stock widgets): finnhub.io/register
  • Gemini API key (the Ask Gemini chat, v1.11 — free, no card): aistudio.google.com/app/apikey → paste into Settings → Gemini API key
  • Find your gist ID: open the dashboard → Settings (⚙) → Sync → Gist ID

One-time setup (already done)

The project is deployed and the first browser is set up. To repeat this whole sequence on a fresh machine, see README.md.

git init && git add . && git commit -m "Initial StartDashboard"
git remote add origin https://github.com/TECGUNTHER/startdashboard.git && git push -u origin main

Then in GitHub: Settings → Pages → Deploy from a branch → main / (root).

To launch

Open in any browser:

open https://tecgunther.github.io/startdashboard/

For local-only preview (no sync), just double-click index.html.

No port required — static page served by GitHub Pages.

Troubleshooting

  • "Bad credentials" on first run · PAT was pasted wrong or is missing the gist scope. Regenerate.
  • Sync indicator turns red · Click it to reload from the gist. If still failing, the token may have expired — regenerate and reset this browser in Settings.
  • Bookmarklet popup blocked · Allow popups for tecgunther.github.io in browser settings.
  • Changes not appearing across browsers · Reload the other browser. Sync polls every ~60s; reload is instant.
  • Markets / Watchlist say "Add your Finnhub API key" · As of v1.9, stock widgets use Finnhub, not Twelve Data. Generate a free key at finnhub.io/register and paste it into Settings. The old Twelve Data key is ignored.
  • "Rate limit reached" on Markets / Watchlist · Finnhub’s free tier is 60 calls/min. The widget backs off and auto-refreshes once after 60 seconds — no action needed.
  • Ask Gemini chat says "Add your free Gemini API key" · No key set. Generate one free at aistudio.google.com/app/apikey and paste it into Settings → Gemini API key. The link-out button still works without a key.
  • Chat says "Couldn’t reach Gemini" · Either a network hiccup, a rate limit, or the browser blocked the direct call (CORS). Use the "Open full Gemini" link in the note to continue in your real Gemini account.
